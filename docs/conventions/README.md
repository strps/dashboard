# Conventions

Day-to-day rules for writing code in this repository. These reflect patterns
already present in the codebase — when in doubt, grep for an existing example and
match it.

- [naming.md](naming.md) — file, folder, and symbol naming.
- [code-style.md](code-style.md) — TypeScript, imports, the `"use client"` boundary, `server-only`.
- [styling.md](styling.md) — Tailwind v4, dark-mode defaults, component composition.
- [git.md](git.md) — commit message conventions.

For architectural rules (where code *lives* and how subsystems fit together) see
[../standards/](../standards/).
